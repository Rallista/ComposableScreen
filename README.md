# ComposableScreen

Render [Jetpack Compose](https://developer.android.com/jetpack/compose) content onto the custom map
surface of an [AndroidX Car App](https://developer.android.com/training/cars/apps) `Screen` — for
both Android Auto and Android Automotive OS.

The Car App library hands your app a raw `Surface` for the map area but no view system on top of it.
`ComposableScreen` hosts that surface on a virtual display and presents a `ComposeView` into it, so
you can write your map/overlay UI as ordinary `@Composable` functions while still returning a normal
Car App `Template` for the rest of the screen.

## Installation

```kotlin
dependencies {
  implementation("io.github.rallista:car-app-compose:<version>")
}
```

## Documentation

Usage, required permissions, and surface gesture handling are documented in the API reference
generated by [Dokka](https://kotlinlang.org/docs/dokka-introduction.html):

```bash
./gradlew :composable-screen:dokkaGenerate
```

The module overview and examples live in
[`composable-screen/dokka/module.md`](composable-screen/dokka/module.md).
